# REME — архитектура MVP

## Цель

REME — социальная платформа для людей, которые публикуют свою жизнь, находят людей рядом и монетизируют свою страсть.

Слоган: «Смотри тех, кого выбрал. Находи новых людей рядом. Публикуй своё. Монетизируй свою страсть.»

## Первый MVP

1. Лента публикаций.
2. Рекомендации и люди рядом.
3. Создание публикации.
4. Личные сообщения.
5. Профиль автора.
6. Базовые действия: лайк, комментарий, репост, донат.

## Навигация

- Лента
- Рекомендации
- Создать
- Чаты
- Профиль

На мобильном экране навигация располагается снизу. На широком экране она превращается в компактную боковую панель.

## Принципы

- Контент — главный элемент интерфейса.
- Минимум лишних экранов и вложенных меню.
- Вертикальная лента с фото, видео и форматом «До / После».
- Рекомендации не смешиваются с основной лентой.
- Любое действие должно быть доступно за один-два нажатия.

## Технологическая база

- Next.js App Router
- TypeScript
- CSS Modules / обычный CSS на первом этапе
- Позже: PostgreSQL, object storage, auth и realtime messaging

## Этапы

### Этап 1 — визуальный прототип

- Первый экран ленты.
- Нижняя навигация.
- Карточка публикации.
- Состояния пустой ленты и загрузки.

### Этап 2 — данные

- Авторизация.
- Профили.
- Публикации и реакции.
- Загрузка медиа.

### Этап 3 — социальный граф

- Подписки.
- Рекомендации.
- Комментарии и чаты.

### Этап 4 — монетизация

- Донаты.
- Платные публикации.
- Поддержка авторов.
