# REME — архитектура нативного Apple MVP

## Цель

REME — социальная платформа для людей, которые публикуют свою жизнь, находят людей рядом и монетизируют свою страсть.

Слоган: «Смотри тех, кого выбрал. Находи новых людей рядом. Публикуй своё. Монетизируй свою страсть.»

## Платформа

- Нативное приложение Apple на SwiftUI.
- Основная цель первого прототипа: iPhone.
- Архитектура сразу готова к iPad и macOS через multiplatform target.
- Минимальная версия: iOS 18+.

## Первый MVP

1. Лента публикаций.
2. Рекомендации и люди рядом.
3. Создание публикации.
4. Личные сообщения.
5. Профиль автора.
6. Лайк, комментарий, репост и донат.

## Навигация

Главный контейнер — TabView с пятью вкладками:

- Лента
- Рекомендации
- Создать
- Чаты
- Профиль

На iPhone используется нижняя системная навигация Apple. Не ломаем привычную пользователю раскладку.

## Архитектура приложения

- SwiftUI для интерфейса.
- MVVM для экранов.
- Swift Observation для состояния.
- SwiftData для локального прототипа.
- Async/await для сетевого слоя.
- Дальше: API, авторизация, object storage и realtime messaging.

## Этапы

### Этап 1 — визуальный прототип

- Root TabView.
- Первый экран ленты.
- Карточка публикации с фото/видео.
- Формат «До / После» как два медиа-блока в одной публикации.
- Кнопки лайка, комментария, репоста и доната.
- Нижняя навигация.

### Этап 2 — локальные данные

- Модели User, Post, Media и Comment.
- SwiftData.
- Создание публикаций из PhotosPicker.
- Локальные реакции.

### Этап 3 — backend

- Авторизация.
- Профили и подписки.
- Публикации и медиа.
- Рекомендации.
- Чаты.

### Этап 4 — монетизация

- Донаты.
- Платные публикации.
- Поддержка авторов.
